# Roadmap Petanque-Z

Приложение для игроков в петанк: помогает определить, какой шар ближе к кошонету, и вести счёт игры.

Разработка разбита на три фазы. В конце каждой фазы ставится git-тег — это позволяет откатиться к рабочему состоянию одной командой:

```bash
git checkout phase1-camera   # откат к концу Phase 1
git checkout phase2-arkit    # откат к концу Phase 2
git checkout phase3-scoring  # откат к концу Phase 3
```

---

## Phase 1 — Камера + детекция шаров

**Цель:** живой видеопоток с камеры с наложенными bbox шаров и кошонета в реальном времени.

**Технологии:** AVFoundation (захват), Core ML (инференс YOLOv10n).

**Критерии готовности:**
- [ ] Приложение запускается и запрашивает доступ к камере
- [ ] Показывает живой видеопоток задней камеры
- [ ] В реальном времени рисует bbox вокруг шаров и кошонета
- [ ] Подписывает класс и процент уверенности
- [ ] Показывает FPS и количество детекций
- [ ] Собирается без ошибок, unit-тесты зелёные
- [ ] Запушено на GitHub, тег `phase1-camera`

**Тег:** `phase1-camera`

---

## Phase 2 — ARKit и измерение расстояний

**Цель:** измерить расстояние от каждого шара до кошонета в сантиметрах, даже если шары не влезают в один кадр (пользователь передвигает камеру — позиции накапливаются в мировой системе координат ARKit).

**Технологии:** ARKit (`ARSession`, мировая система координат, raycast в найденную плоскость земли), Core ML-детекция из Phase 1.

**План:**
- Привязать 2D-детекции к 3D-миру через raycast центра bbox в плоскость
- Хранить мировые позиции шаров/кошонета (накапливать по мере движения камеры)
- Показывать расстояние в см от каждого шара до кошонета
- Показать ранжирование шаров по близости
- Отображать «диаметр» шара (≈7.1 см для стандарта) для визуального масштаба

**Критерии готовности:**
- [ ] Расстояния отображаются в см поверх каждого шара
- [ ] Шары накапливаются по мере движения камеры
- [ ] Шары сортируются по близости к кошонету
- [ ] Точность задокументирована и измерена на устройстве

**Тег:** `phase2-arkit`

**⚠️ Ограничение (без LiDAR):** ожидаемая точность ±5–15 см. Граничные случаи (шары на 2–3 см друг от друга) могут определяться неверно.

## Phase 3 — Команды, шары и счёт

**Цель:** зарегистрировать команды и их шары (embedding), автоматически определять владельца шара и вести счёт игры.

**Технологии:** MobileNet (или похожая) для эмбеддингов, идентификация шаров по визуальным признакам (цвет, рисунок, царапины), логика счёта петанка.

**План:**
- UI регистрации команд (название, цвет)
- Регистрация шаров команды — съёмка с нескольких ракурсов (вращение шара), сохранение эмбеддинга
- Сопоставление детектированного шара с базой эмбеддингов (косинусное расстояние)
- Подсчёт очков по правилам петанка (ближайший шар + все шары одной команды до первого шара соперника)
- UI ведения счёта по фреймам

**Критерии готовности:**
- [ ] Можно зарегистрировать две команды
- [ ] Шары регистрируются и опознаются в кадре
- [ ] Счёт считается автоматически после измерения дистанций
- [ ] Ведётся история фреймов

**Тег:** `phase3-scoring`

**⚠️ Ограничение:** Re-идентификация *отдельных* шаров одной команды очень сложна (металлические сферы выглядят одинаково). Реалистично отличать команды (если шары визуально разные) — отличать шары одного игрока может оказаться невозможным. Тестируется на реальных шарах.
